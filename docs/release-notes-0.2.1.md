# GameSave Manager 0.2.1

`0.2.1` 是安全自动更新验证版，主要用于验证已安装 `0.2.0` 的客户端能否发现、下载、校验并覆盖安装本版本，以及启动失败时能否回滚到上一版本。

## 本版变化

- 版本升级到 `0.2.1`，继续使用 `0.2.0` 已固定的 RSA 3072 位自签发布者证书和独立 ECDSA 更新清单密钥。
- 修复 GitHub Actions 自签发布过程中的证书验证阻塞问题，不再修改 Runner 根证书库。
- 固定并校验 Inno Setup 简体中文语言文件来源，保证安装器构建可复现。
- 严格接受且仅接受固定自签根不受信这一种发布环境链错误，同时继续校验叶证书指纹、文件摘要和可信时间戳。
- 清理成功处理预期 SignTool 结果后的退出码，避免已完成验签的发布任务被错误判定为失败。

## 从 0.2.0 验证自动更新

1. 确认当前已安装版本为 `0.2.0`，发布者证书仍位于当前用户的“受信任的根证书颁发机构”和“受信任的发布者”。
2. 打开客户端“设置”页面，确认已允许接收预发布更新，然后点击“检查更新”。
3. 确认客户端显示 `0.2.1`，下载过程中进度可见且可以取消。
4. 下载完成后执行安装，客户端应退出并由更新引导程序完成覆盖升级。
5. 重新启动后确认设置页面版本为 `0.2.1`，原有登录状态、本地游戏配置和存档同步状态仍然存在。

如果客户端没有发现更新，请先确认 GitHub Release 已发布为预发布、系统时间正确，并且可以访问 GitHub Release 及其附件。

## 全新或手动安装

未安装 `0.2.0` 的用户仍可从官方 GitHub Release 手动下载 `0.2.1`。首次安装必须先按照仓库说明核对并安装公开发布者证书，再运行：

```text
GameSaveManager-Setup-0.2.1.exe
```

发布者证书 SHA-256 固定指纹为：

```text
14768BC7D3CF2B1EB5BBE3228ADC8A3D35A1F923CB806B64147D7CFD3BCA8E35
```

## 验收说明

本次自动化发布会校验版本、标签、程序集、Authenticode 签名、可信时间戳、安装包摘要、GitHub 资产摘要、`SHA256SUMS.txt` 以及独立签名更新清单。真实客户端内升级、启动确认和故障回滚仍需要在已安装 `0.2.0` 的 Windows 环境中人工验收。
