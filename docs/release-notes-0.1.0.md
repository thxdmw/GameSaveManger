# GameSave Manager 0.1.0

这是 GameSave Manager 的首个公开测试版本，提供 Windows 游戏存档的本地配置、云端快照、跨设备恢复和退出后自动备份能力。

## 主要功能

- 六步添加向导，支持 Steam、Epic、GOG 和本地 EXE/LNK。
- 启动入口测试、工作目录与参数配置，以及实际游戏进程识别。
- Ludusavi Manifest、常见目录、手动选择和运行学习四类存档位置识别方式。
- 多存档根目录、Include/Exclude Glob 和 HKCU 注册表存档。
- SHA-256 内容寻址、缺失对象上传和不可变云端快照。
- 立即备份、退出后自动备份、进度显示、取消、重试和结果摘要。
- 云端时间线、恢复预览、ZIP 导出、历史快照删除和保留策略。
- 安全恢复 staging、原存档安全备份、恢复 Journal 和启动恢复。
- 多设备 HEAD 冲突阻断，以及恢复云端或保留本机两个处理方向。
- 设备与容量管理、深浅主题、开机启动和系统托盘后台运行。

## 系统要求

- Windows 10 或 Windows 11，64 位。
- 可访问已正确部署的 GameSave CMS 服务端。
- 远程服务端必须使用 HTTPS；HTTP 只允许本机回环地址。

安装包为 `win-x64` 自包含版本，普通用户不需要另外安装 .NET Runtime。

## 安装与升级

1. 下载 `GameSaveManager-Setup-0.1.0.exe` 和 `SHA256SUMS.txt`。
2. 对安装包计算 SHA-256，并与校验文件比较。
3. 运行安装包，按向导完成当前 Windows 用户安装。
4. 首次启动后，在“账户”页面填写服务端地址并登录。

当前版本尚未配置 Authenticode 代码签名。公开分发前应完成签名；未签名安装包可能触发 Windows SmartScreen 提示。

## 首次使用建议

- 先为测试游戏创建一次手动备份并检查时间线。
- 执行一次恢复预览，确认快照文件数量和大小符合预期。
- 重要存档首次恢复前，额外保留游戏原始存档副本。
- 需要退出后自动备份时，关闭窗口请选择“最小化到托盘”。

## 已知限制

- 尚未提供客户端内自动更新，需要手动下载并运行新版本安装包。
- 尚未完成代码签名，因此安装时可能显示未知发布者。
- 自动存档识别结果仍需用户预览并确认，客户端不会自动采用候选目录。

## 完整性校验

正式发布时以同目录 `SHA256SUMS.txt` 中的值为准。每次重新生成安装包都会改变 SHA-256，不能沿用旧校验值。
