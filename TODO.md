# 后续任务清单

本文只记录当前客户端仍需完成的工作。已经落地的游戏发现、六步添加向导、多根目录、注册表存档、手动/自动备份、时间线、安全恢复、设备管理和主题功能不再重复列为待办。

## P0：发布前验收

- [ ] 在真实 CMS 与对象存储环境完成“首次备份、相同内容重复备份、增量备份、取消、断网、服务端 5xx、对象上传失败”端到端验收。
- [ ] 使用至少两台 Windows 设备验证 HEAD 冲突的两条路径：恢复云端版本、保留本机并创建新版本；确认任何路径都不会静默覆盖另一端存档。
- [ ] 对安全恢复执行故障注入，覆盖对象下载中断、磁盘空间不足、staging 构建失败、目录切换中断、文件最终校验失败和应用重启恢复。
- [ ] 对文件与注册表混合恢复执行交叉故障注入，覆盖注册表应用失败、文件已切换但注册表未应用、回滚失败和需要人工介入的现场。
- [ ] 使用真实大型存档验证文件数、单文件大小、总容量、长路径、只读文件、被占用文件和权限不足等边界。
- [ ] 安装、升级、覆盖安装和卸载发布包，确认 SQLite、日志、恢复 Journal、内容缓存和 Credential Manager 凭据的保留或清理行为符合产品约定。
- [ ] 验证应用最小化到托盘、完全退出、开机启动和多游戏进程监听在长时间运行后的稳定性。

## P1：同步可靠性与可诊断性

- [ ] 统一登录失效、请求超时、限流、服务端 5xx、断网和对象存储失败的用户提示、退避策略与“立即重试”入口。
- [ ] 在冲突对话框中补充本机与远端快照的时间、设备、文件数量和大小对比，进一步明确两个选择的后果。
- [ ] 增加可控故障注入入口或测试替身，避免恢复和同步异常只能依赖人工制造。
- [ ] 为本地 SQLite 数据库损坏、迁移失败和备份恢复建立自动化验证。
- [ ] 为后台自动备份增加连续失败通知与抑制策略，避免托盘状态下失败不可见或重复提示。

## P2：自动化与发布工程

- [ ] 将所有只读 ViewModel 属性的显示绑定纳入自动检查，防止误用双向绑定导致页面加载失败。
- [ ] 扩展 WPF 冒烟验证，覆盖主题切换、对话框打开、折叠区域展开和主要页面命令可用性。
- [ ] 在 CI 中增加 Release 发布物启动验证和安装包静默安装/卸载验证。
- [ ] 接入 Windows 代码签名与可信时间戳，并记录证书轮换流程。
- [ ] 设计签名更新清单、HTTPS 更新源、失败回滚和版本降级策略。

## P3：产品体验

- [ ] 为首次使用增加可跳过的引导，串联登录、添加游戏、确认存档和首次备份。
- [ ] 在时间线增加按游戏、设备、时间和备份类型筛选，以及大列表分页/虚拟化。
- [ ] 在设备页补充当前设备标识、最后同步游戏和撤销后的影响说明。
- [ ] 为本地缓存与日志提供容量查看、安全清理和导出诊断包入口。
- [ ] 增强无障碍支持：键盘导航、焦点样式、屏幕阅读器名称和高对比度检查。

- [x] 游戏启动安全、进程识别、添加向导状态隔离和存档候选校验。
- [x] 注册表预览、恢复 Journal 启动恢复和游戏详情同步状态验证。
