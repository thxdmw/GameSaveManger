# 更新记录

## 未发布

## 0.3.2 - 2026-07-21

- 修复删除游戏时服务端删除目标与客户端列表移除目标可能不一致的严重问题；删除命令现在必须携带并始终使用确认时的游戏 ID。
- 切换游戏时立即清空存档候选、路径输入、预览统计、运行学习状态与同步摘要，防止复用上一游戏的临时数据。
- 本地配置保存、快照恢复/预览/导出/删除、保留策略、Manifest 扫描和自动同步启停均固定操作开始时的游戏或快照 ID；迟到的异步结果不得覆盖新选择。
- 修复撤销设备操作存在的同类目标漂移风险，确认后始终撤销明确的设备 ID。
- 增加删除目标、设备目标、游戏切换输入清理及同步缓存隔离回归验证。

## 0.3.1 - 2026-07-21

- 修复添加游戏向导只有“运行游戏学习”、缺少“完成学习”和“取消学习”的问题。
- 学习开始后禁止重复开始，避免覆盖游戏运行前的文件基线；完成、取消或启动失败时会正确释放学习状态。
- 在存档检测步骤直接显示学习状态说明，明确提示用户保存并退出游戏后完成学习。
- 增加学习命令状态回归验证；本版本暂不调整自动检测把单文件规则扩大为游戏根目录的问题。

## 0.3.0 - 2026-07-21

- 备份快照记录可移植的存档路径模板、路径来源和扫描规则，并在时间线展示备份路径。
- 新电脑添加相同游戏时可参考云端历史路径；候选目录仍须在本机存在、预览并由用户确认。
- 从客户端启动游戏前检查云端 HEAD：本机未修改时安全拉取最新进度，本机与云端均变化时阻止启动并进入冲突处理。
- 将设备标识迁移到 Windows 凭据管理器，删除客户端 SQLite 或 AppData 后仍可识别为同一设备。
- 本机配置与同步 HEAD 按服务端、账号和游戏严格隔离，退出或凭据失效时停止旧账号的自动同步。
- 手动和自动备份的完成、失败、取消与冲突结果会通过 Windows 托盘通知显示。
- 本地数据库升级到 schema 8；配套服务端需先部署账号会话接口和 `V9__gamesave_snapshot_roots.sql`。

## 0.2.1 - 2026-07-20

- 发布首个用于验证客户端内安全自动更新与失败回滚链路的补丁版本。
- 修复自签发布流水线的证书验证、简体中文 Inno Setup 语言文件获取以及预期验签退出码处理。
- 继续使用 `0.2.0` 已固定的发布者证书和更新清单密钥，验证跨版本签名信任连续性。

## 0.2.0 - 2026-07-20

- 增加启动后后台检查与手动检查客户端更新、下载进度和取消操作。
- 增加独立 ECDSA 签名更新清单、GitHub 资产摘要、`SHA256SUMS.txt`、本地 SHA-256 与 Windows 发布者签名的完整校验链。
- 增加独立更新引导程序、启动健康确认和升级失败自动回滚。
- 采用 RSA 3072 位自签代码签名证书，提供公开证书、指纹元数据以及安装和移除信任脚本；发布私钥始终保存在仓库外。
- 增加不可覆盖的 GitHub 预发布工作流，自动签名主程序、更新引导程序、安装包和卸载程序，并生成完整发布校验文件。
- 统一程序集、发布脚本、安装包、设置页和 GitHub 标签的版本来源。
- 完善添加游戏状态隔离、游戏详情切换、备份进度、存档配置、卡片菜单与客户端统一对话框等交互。

## 0.1.0 - 2026-07-17

- 完成游戏发现、启动配置、手动备份、退出后自动同步和多设备冲突处理。
- 增加多存档根目录、HKCU 注册表存档、时间线恢复与本机安全回滚。
- 增加重解析点隔离、扫描预算、凭据安全存储及发布包完整性校验。
