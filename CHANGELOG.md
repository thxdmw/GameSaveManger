# 更新记录

## 未发布

## 0.3.5 - 2026-07-22

- 修复已选中一款游戏时打开添加向导，新游戏草稿的 EXE、存档目录和启动配置可能提前写入旧游戏 `local_game_profile` 的问题；向导现在使用独立草稿，创建接口返回新 `gameId` 前拒绝任何本机配置写入。
- 向导中的主存档目录确认、附加目录增删和 EXE 选择只更新草稿；取消向导后完整恢复原游戏选择、配置和自动同步状态。
- 手动选择存档目录后会立即在当前检测步骤下方显示并选中；再次选择会替换旧的手动候选，不会堆积过期路径。
- 修复窗口 `Closing` 阶段再次调用 `Show`、`Hide` 或 `Close` 导致“退出前清理失败”的问题；退出确认与实际关闭改为事件返回后执行，退出期间不再弹出冲突或操作确认窗口。
- 新增添加第二款游戏零写入、底层归属防线、手动目录即时回显和 WPF 延后关闭回归验证。本版本不修改服务端协议或本地数据库结构，也不自动改写已有本机配置。

## 0.3.4 - 2026-07-22

- 修复切换游戏后详情页可能展示另一款游戏 EXE、监控进程和存档目录的问题；选择游戏时按服务端、账号和游戏 ID 同步恢复对应本机配置，“管理存档”会等待选择加载完成后再进入详情。
- 立即备份、重试和“保留本机版本”只读取操作开始时已确认的持久化配置，不再把界面瞬态字段写回 SQLite；界面配置与持久化记录不一致时直接停止同步，避免扫描错误目录并产生假冲突。
- 添加向导、账号切换和异步游戏加载使用独立上下文代次，旧请求不能在向导打开或游戏切换后覆盖当前配置；所有配置保存同时校验当前界面归属。
- Steam 与 Epic 配置不再追加切换期间残留的其他游戏进程；平台 EXE 越出当前安装目录，或 EXE 与存档目录同时复制自另一游戏时，会显示配置异常并禁用启动、手动备份和自动同步。
- 新增游戏配置切换、污染配置拦截、平台进程隔离以及同步零写入回归验证。本版本不修改服务端协议，也不新增本地数据库迁移。

## 0.3.3 - 2026-07-22

- 新建游戏完成后立即保留已确认的存档配置，可直接执行首次备份；删除游戏、设备、快照和保留策略操作固定使用确认时的明确 ID，并等待同游戏同步任务安全退出。
- 同步、恢复、导出和游戏启动按游戏串行，账号或选择变化后拒绝迟到结果；启动游戏前检查云端 HEAD，本机未修改时自动恢复远端进度，双方均修改时阻止启动并进入冲突处理。
- 完善多根目录与注册表存档的事务恢复、空目录精确恢复、安全备份和崩溃 Journal；应用存档前再次确认游戏未运行，恢复完成后的清理失败不再误触发回滚。
- 正式备份始终重新读取文件并计算 SHA-256，服务端返回的游戏、设备、配额、策略、快照和完整 Manifest 均校验身份、唯一性、路径及协议上限，写请求响应不确定时通过云端事实核对结果。
- 修复运行游戏学习丢失新建具体子目录的问题；Ludusavi 明确标记为配置文件的单文件规则不再把整个游戏安装目录误判为存档目录。
- 本机配置和同步 HEAD 按服务端、账号、游戏三级隔离，损坏配置停止加载；同一台 Windows 电脑使用基于 MachineGuid 的稳定设备身份，清除客户端数据后不再重复登记设备。
- 手动与自动备份提供 Windows 成功、失败、取消和冲突通知；账号切换会清空输入、搜索、冲突和快捷方式缓存，本地对象缓存、更新、恢复及清理流程增加磁盘空间、路径越界和重解析点保护。
- 本地数据库升级到 schema 9，并新增覆盖删除目标、会话竞争、破坏性同步、提交歧义、精确恢复、Hash 缓存、设备身份、运行学习和云端响应校验的回归验证。

## 0.3.2 - 2026-07-21

- 修复删除游戏时服务端删除目标与客户端列表移除目标可能不一致的严重问题；删除命令现在必须携带并始终使用确认时的游戏 ID。
- 切换游戏时立即清空存档候选、路径输入、预览统计、运行学习状态与同步摘要，防止复用上一游戏的临时数据。
- 本地配置保存、快照恢复/预览/导出/删除、保留策略、Manifest 扫描和自动同步启停均固定操作开始时的游戏或快照 ID；迟到的异步结果不得覆盖新选择。
- 修复撤销设备操作存在的同类目标漂移风险，确认后始终撤销明确的设备 ID。
- 增加删除目标、设备目标、游戏切换输入清理及同步缓存隔离回归验证。

## 0.3.1 - 2026-07-21

- 修复添加游戏向导只有“运行游戏学习”、缺少“完成学习”和“取消学习”的问题。
- 学习开始后禁止重复开始，避免覆盖游戏运行前的文件基线；完成、取消或启动失败时会正确释放学习状态。
- 在存档检测步骤直接显示学习状态说明，明确提示用户保存并退出游戏后完成学习。
- 增加学习命令状态回归验证；本版本暂不调整自动检测把单文件规则扩大为游戏根目录的问题。

## 0.3.0 - 2026-07-21

- 备份快照记录可移植的存档路径模板、路径来源和扫描规则，并在时间线展示备份路径。
- 新电脑添加相同游戏时可参考云端历史路径；候选目录仍须在本机存在、预览并由用户确认。
- 从客户端启动游戏前检查云端 HEAD：本机未修改时安全拉取最新进度，本机与云端均变化时阻止启动并进入冲突处理。
- 将设备标识迁移到 Windows 凭据管理器，删除客户端 SQLite 或 AppData 后仍可识别为同一设备。
- 本机配置与同步 HEAD 按服务端、账号和游戏严格隔离，退出或凭据失效时停止旧账号的自动同步。
- 手动和自动备份的完成、失败、取消与冲突结果会通过 Windows 托盘通知显示。
- 本地数据库升级到 schema 8；配套服务端需先部署账号会话接口和 `V9__gamesave_snapshot_roots.sql`。
